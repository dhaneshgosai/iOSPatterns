MVC

iOS natural UI architecture is MVC (Model, View, Controller).

An app is structured in 3 classes of components:

  • models (persistent data, backend service, business rules, etc...)
  • views (UI elements user sees and interacts with)
  • view controllers (tie everything together)

We need

  • models <-> View controllers communication
  • View controllers <-> views communication
  • View Controllers <-> View Controllers communication
  • models <-> models communication

We Want

  • Lose coopling
  • Separation of Concerns
  • Reusability
  • Maintainability
  • To do the RIGHT thing

## Delegation

A delegate is an object that acts on behalf of, or in coordination with, another object when that object encounters an event in a program.

  • The only thing the notifier object knows about its delegate is that is implements a shared protocol.
  • The notifier typically interacts with its delegate via a (weak) property.
  • The delegate object usually owns the notifier object (it hols a strong reference to it, it even created it).

Example of a shared protocol between the delegate and the delegator


@protocol MyObjectDelegate<NSObject>

@required
// object notifies delegate of an event:
-(void) myObject:(MyObject *)myObject didFinishSomething:(int)result;
// object requires information from its delegate:
-(BOOL) myObject:(MyObject *)myObject pleaseTellMeIf:(int)blah;

@optional
// object notifies delegate of another (optional) event:
-(void) myObject:(MyObject *)myObject madeProgress:(float) percents;

@end

The notifying object:


@interface MyObject

-(void) doSomeTask;

@property (nonatomic, weak) id<MyObjectDelegate> delegate;

@end


@implementation MyObject

-(void) doSomeTask
{
	// compute stuff ...
	//...

	if ([self.delegate respondsToSelector:
					@selector(myObject:MadeProgress:)]) {
		// half way done
		[self.delegate myObject:self madeProgress:0.5]; 
	}

	// do work
	// ...

	[self.delegate myObject:self didFinishSomething:123];
}
@end

The delegate


@interface ObjectBid<MyObjectDelegate>

@property (nonatomic, strong) MyObject *myObject;

@end

@implementation ObjectB

-(void)viewDidLoad
{
	self.myObject = [[MyObject alloc] init];
	self.myObject.delegate = self;
	[self.myObject doSomeTask];
}

-(void) myObject:(MyObject *)myObject didFinishSomething:(int)result
{
	// take action.
}

// object requires information from its delegate:
-(BOOL) myObject:(MyObject *)myObject pleaseTellMeIf:(int)blah
{
	return YES;
}
@end
