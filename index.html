MVC

iOS natural UI architecture is MVC (Model, View, Controller).

An app is structured in 3 classes of components:

  • models (persistent data, backend service, business rules, etc...)
  • views (UI elements user sees and interacts with)
  • view controllers (tie everything together)

We need

  • models <-> View controllers communication
  • View controllers <-> views communication
  • View Controllers <-> View Controllers communication
  • models <-> models communication

We Want

  • Lose coopling
  • Separation of Concerns
  • Reusability
  • Maintainability
  • To do the RIGHT thing

## Delegation

Shared Protocol between 2 objects


@protocol BigCalculatorDelegate<NSObject>

@required
-(void) bigCalculator:(BigCalculator1 *)calculator 
      didFinishOperationWithResult:(float)result;

@optional
-(void) bigCalculator:(BigCalculator1 *)calculator 
   madeProgressOnCalculation:(float)percentProgress;

@end

Defines method the delegate must implement.

A property to call the delegate


@property (nonatomic, weak) id<BigCalculatorDelegate> delegate;

When the object needs somthing from its delegate:


[self.delegate bigCalculator:self didFinishOperationWithResult:1.618];
